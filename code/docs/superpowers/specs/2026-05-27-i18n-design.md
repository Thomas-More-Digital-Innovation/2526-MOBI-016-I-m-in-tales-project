# i18n Design: English + Dutch with Language Switcher

**Date:** 2026-05-27
**Status:** Approved

## Overview

Add full English/Dutch internationalisation to the app using the already-installed `typesafe-i18n` library (React adapter). A language switcher in the Header lets users toggle between the two locales; the choice is persisted to `localStorage`.

---

## 1. Locale Setup

- Delete `src/i18n/de/` (auto-generated German example — not needed).
- Create `src/i18n/nl/index.ts` with Dutch translations.
- Populate `src/i18n/en/index.ts` with English translations (replaces the placeholder `HI` key).
- Re-run `pnpm typesafe-i18n` to regenerate `i18n-types.ts` and `i18n-util.ts` with `locales: ['en', 'nl']`.

## 2. App Wiring

- Use typesafe-i18n's `localStorageDetector` to detect the saved locale on startup.
- Load both locales synchronously via the sync loader in `i18n-util.sync.ts` before render.
- Wrap the entire app in `App.tsx` with the `TypesafeI18n` React provider:

```tsx
// App.tsx
import TypesafeI18n from './i18n/i18n-react'
import { detectLocale } from './i18n/i18n-util'
import { localStorageDetector } from 'typesafe-i18n/detectors'
import { loadAllLocales } from './i18n/i18n-util.sync'

loadAllLocales()
const locale = detectLocale(localStorageDetector)

function App() {
  return (
    <NfcProvider>
      <TypesafeI18n locale={locale}>
        <RouterProvider router={router} />
      </TypesafeI18n>
    </NfcProvider>
  )
}
```

Any component gets translations via:
```tsx
const { LL, setLocale } = useI18nContext()
```

---

## 3. Language Switcher Component

**File:** `src/app/components/LanguageSwitcher.tsx`

- Uses `useI18nContext()` to read `locale` and call `setLocale`.
- On locale change, also writes `'lang'` key to `localStorage` so the `localStorageDetector` picks it up on next launch.
- Renders two compact text buttons **EN** and **NL** styled like existing `HeaderButton` components.
- The active locale's button is visually highlighted (brighter background or underline).
- Placed directly inside `Header.tsx`'s right-side button row — no per-page changes needed.

---

## 4. String Replacement

Every component with hardcoded UI text is updated to call `const { LL } = useI18nContext()` and replace literals with `LL.KEY()`:

| File | Strings replaced |
|------|-----------------|
| `src/app/page.tsx` | 5 button labels + 4 tooltips |
| `src/app/manageStory/page.tsx` | title, loading, empty state, button labels, dialog strings |
| `src/app/makeStory/page.tsx` | 2 conditional header titles |
| `src/app/storyOverview/page.tsx` | title, tooltip, loading, empty state, modal labels |
| `src/app/help/page.tsx` | page heading |
| `src/app/components/Header.tsx` | LanguageSwitcher added to right button row |

---

## Constraints

- No loading screen needed: both locale files are small and loaded synchronously at startup.
- `localStorageDetector` from typesafe-i18n reads the `lang` key; `setLocale` + manual `localStorage.setItem('lang', locale)` writes it on switch.
- The `de/` locale folder is deleted entirely; only `en/` and `nl/` exist after this work.
